# Security policy

Do not report exploitable vulnerabilities, credentials, private data, or sensitive deployment details in public issues or Discussions.

Use GitHub private vulnerability reporting for this repository. Until a dedicated security address exists, that is the canonical confidential channel under R8-070.

Include the affected release or component, impact, reproduction conditions, evidence, and any known mitigations. Do not include unrelated sensitive payloads.

The project may impose temporary fail-closed restrictions while a report is evaluated. Coordinated disclosure timing will be based on risk, remediation readiness, and affected-party safety.

GKOS v0.76 is a pre-standard and does not constitute a security certification or assurance that any implementation is safe.
