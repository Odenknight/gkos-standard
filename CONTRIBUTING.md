# Contributing to GKOS

GKOS welcomes criticism, implementation reports, examples, fixtures, and amendment proposals. Participation does not grant authority to bind the standard.

Use GitHub Discussions for exploratory questions and amendment concepts. Use issues for actionable ambiguities, defects, implementation reports, or fixture requests. Use pull requests for concrete replacement text or artifacts. Report security-sensitive information through the private process in `SECURITY.md`.

## Change classes

- Editorial
- Clarification
- Normative compatible
- Breaking
- Constitutional
- Security emergency

A normative proposal must identify affected sections and decisions, evidence, exact language, compatibility impact, security/privacy effects, fixture implications, and rollback or supersession route.

No actor may approve or certify its own proposal.

## Licensing and DCO

Documentation contributions are submitted under CC BY 4.0. Software-oriented contributions are submitted under Apache-2.0.

All commits intended for merge MUST carry a Developer Certificate of Origin 1.1 sign-off using `git commit -s`. The sign-off certifies the contributor has the right to submit the contribution under the applicable license. No CLA is currently required.
